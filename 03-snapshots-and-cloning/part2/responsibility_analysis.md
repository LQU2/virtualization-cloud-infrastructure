In an IaaS model, the cloud provider gives basic computing resources such as hardware, storage, networking, and the virtualization layer. The customer must install the operating system then manage it. They configure firewalls, apply updates, deploy applications. This means the customer has strong control but also more work. If the operating system is misconfigured or missing patches, the issue remains the customer’s responsibility even though the infrastructure runs in the cloud.

With PaaS, more responsibility moves to the provider. The provider manages the operating system and the runtime environment. The customer no longer maintains servers or installs system updates. Their focus shifts to deploying application code and managing data. This reduces operational work but also limits control over the environment.

In SaaS, the provider manages the full system including the application itself. The customer usually manages user accounts and access. The software runs through a browser or client instead of customer managed servers. As cloud services move from IaaS to SaaS, the customer controls less of the system while the provider manages more. The customer role shifts away from infrastructure management toward service use and governance.
